# 思考记录：投资研究 Skill 设计

## 核心思考过程

### 1. 第一性原理出发

**问题**: 投资研究的本质是什么？

**答案**: 
- 不是预测价格，而是理解驱动价格的因素
- 不是堆砌数据，而是建立因果链条
- 不是给出答案，而是提供思考框架

### 2. 信息架构设计

**关键洞察**: 信息价值 = 及时性 × 可信度 × 可操作性

**分层**:
- L1 (毫秒): 订单流、微观结构 → 交易执行
- L2 (分钟): 新闻、公告 → 事件驱动
- L3 (日): 资金流向、情绪 → 战术调整
- L4 (月): 宏观、行业景气 → 战略配置
- L5 (年): 结构变化、技术革命 → 长期布局

### 3. 可信度工程

**核心问题**: 单一数据源不可靠

**解决方案**:
- 多源获取 (Yahoo + 东方财富 + CoinGecko)
- 交叉验证 (价格偏差 < 1%)
- 可信度评分 (官方 > 专业 > 聚合)
- 故障转移 (主源失败 → 备用源)

### 4. 从泛泛到具体

**初始版本**:
> "股票减仓，因为市场下跌"

**问题**: 太泛，没有可操作性

**迭代后**:
> "科技板块承压：XLK -2.06%，VIX 飙升至 29.5，高估值成长股对波动率敏感"

**关键改进**:
- 具体数据支撑
- 传导机制解释
- 行业层面聚焦

### 5. 上下文压缩思考

**问题**: 对话长了之后成本和延迟都高

**解决方案**:
- 分层记忆 (短期详细 + 长期摘要)
- 主动压缩 (>10轮或>4000tokens)
- 智能提取 (保留决策/结果/错误)

**权衡**:
- 保留: 关键决策、用户偏好、错误教训
- 丢弃: 中间尝试、重复信息、已修复问题

## 关键决策记录

| 决策 | 原因 | 结果 |
|-----|------|------|
| 使用 Yahoo Finance 为主 | 免费、稳定、覆盖全球 | ✅ 成功 |
| 排除 VIX 从股指平均 | VIX上涨≠股指上涨 | ✅ 修复计算错误 |
| 新浪财经替代东方财富 | API稳定性 | ✅ 更可靠 |
| 行业ETF分析 | 用户需要具体行业影响 | ✅ 满足需求 |
| Markdown格式存储 | 可读性、版本控制 | ✅ 便于管理 |
