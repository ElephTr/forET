### 1.2 启动时加载的上下文文件

| 文件 | 用途 | 加载时机 |
|------|------|----------|
| `SOUL.md` | Agent 人格与行为准则 | 每次会话 |
| `USER.md` | 用户信息与会话偏好 | 每次会话 |
| `AGENTS.md` | 工作空间约定与红线 | 每次会话 |
| `IDENTITY.md` | Agent 身份信息 | 每次会话 |

| `memory/YYYY-MM-DD.md` | 每日上下文 | 每次会话 |
| `HEARTBEAT.md` | 定时任务清单 | 心跳触发时 |

## 2. 任务接收阶段

### 2.1 消息入站流程

```
用户输入
    │
    ▼
┌─────────────────┐
│  Channel Parser │  ← 解析来源（WebChat/Discord/Telegram等）
└─────────────────┘
    │
    ▼
┌─────────────────┐
│ Metadata Inject │  ← 注入可信元数据（schema, channel, surface）
└─────────────────┘
    │
    ▼
┌─────────────────┐
│ Context Loader  │  ← 加载 Project Context（上述所有md文件）
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  Agent Runtime  │  ← 进入核心处理逻辑
└─────────────────┘
```

### 2.2 接收到的原始消息示例

**用户输入：**
```
你现在是一个创业顾问，帮我生成一份关于"AI Agent 服务"的创业计划书，
要求包含市场分析、竞争分析、商业模式、运营计划和财务预测。
```

**注入的元数据（可信）：**
```json
{
  "schema": "openclaw.inbound_meta.v1",
  "channel": "webchat",
  "provider": "webchat",
  "surface": "webchat",
  "chat_type": "direct"
}
```

**加载的上下文（内存中）：**
- SOUL.md → "Be genuinely helpful, not performatively helpful"
- USER.md → "老板", 时区未知
- AGENTS.md → 工作空间规则、记忆管理规范
- IDENTITY.md → Name: ET, Emoji: 👽

## 3. 意图理解阶段

### 3.1 自然语言理解（NLU）

Agent 对输入进行多层次解析：

| 解析层 | 处理内容 | 示例输出 |
|--------|----------|----------|
| **实体识别** | 提取关键实体 | `主题: AI Agent 服务`, `文档类型: 创业计划书` |
| **意图分类** | 确定用户目标 | `意图: 生成商业文档` |
| **约束提取** | 识别格式/内容要求 | `必需章节: [市场分析, 竞争分析, 商业模式, 运营计划, 财务预测]` |
| **隐式需求推断** | 基于上下文补全 | `语言: 中文`, `详细程度: 完整` |

### 3.2 任务分解

将复杂任务拆分为可执行子任务：

```
生成创业计划书
├── 1. 市场分析
│   ├── 1.1 行业规模与趋势
│   ├── 1.2 目标用户画像
│   └── 1.3 需求验证
├── 2. 竞争分析
│   ├── 2.1 竞争对手识别
│   ├── 2.2 竞品优劣势对比
│   └── 2.3 差异化定位
├── 3. 商业模式
│   ├── 3.1 价值主张
│   ├── 3.2 收入模式
│   └── 3.3 成本结构
├── 4. 运营计划
│   ├── 4.1 启动阶段（0-6月）
│   ├── 4.2 增长阶段（6-18月）
│   └── 4.3 规模化阶段（18月+）
└── 5. 财务预测
    ├── 5.1 启动资金需求
    ├── 5.2 收入预测（3年）
    └── 5.3 盈亏平衡分析
```

## 5. 执行规划阶段

### 5.1 执行策略选择

根据任务复杂度选择执行模式：

| 模式 | 适用场景 | 本任务选择 |
|------|----------|-----------|
| **单步执行** | 简单查询、直接回答 | ❌ |
| **多步规划** | 复杂任务、需多工具协作 | ✅ |
| **子代理委派** | 耗时任务、需隔离执行 | ❌ |

### 5.2 工具链规划

确定需要调用的工具序列：

```
Step 1: web_search
  └─ 搜索 "AI Agent 市场规模 2026"
  └─ 搜索 "AI Agent 创业公司 融资"
  
Step 2: web_search  
  └─ 搜索 "AI Agent 商业模式 B2B"
  
Step 3: file_write
  └─ 将结果写入创业计划书.md
  
Step 4: exec
  └─ git add / commit / push
```

## 6. 工具调用阶段

### 6.1 工具调用协议

每个工具调用遵循标准格式：

```json
{
  "tool": "web_search",
  "parameters": {
    "query": "AI Agent 市场规模 2026",
    "count": 5,
    "freshness": "year"
  },
  "purpose": "获取AI Agent市场最新数据",
  "expected_output": "市场规模的量化数据"
}
```

### 6.2 实际调用序列示例

**调用 1: 市场数据搜索**
```python
# 内部工具调用
web_search(
    query="AI Agent 市场规模 2026 预测",
    count=5,
    freshness="year"
)
```

**返回结果：**
```json
{
  "results": [
    {
      "title": "全球AI Agent市场规模预测报告",
      "url": "https://example.com/report",
      "snippet": "2026年全球AI Agent市场规模预计达到150亿美元，年复合增长率45%..."
    },
    ...
  ]
}
```

**调用 2: 竞争情报搜索**
```python
web_search(
    query="AI Agent 创业公司 融资 2025 2026",
    count=5
)
```

**调用 3: 商业模式搜索**
```python
web_search(
    query="AI Agent 商业模式 SaaS B2B 定价",
    count=5
)
```

## 附录：技术细节

### A. 系统提示词（System Prompt）结构

```
[身份定义]
你是 OpenClaw 中的 AI Agent，运行模型 moonshot/kimi-k2.5

[核心指令]
- 使用可用工具完成任务
- 优先读取技能文件再执行
- 保持简洁，避免填充词

[上下文文件]
- SOUL.md: 人格定义
- USER.md: 用户信息
- AGENTS.md: 工作空间规则
- IDENTITY.md: Agent身份

[工具使用规则]
- 工具名称大小写敏感
- 优先使用专门工具而非通用方法
- 批量操作优于循环调用

[回复规范]
- 使用 [[reply_to_current]] 回复当前消息
- HEARTBEAT_OK 用于心跳确认
- NO_REPLY 用于无需回复场景
```

### B. 内存管理策略

| 内存类型 | 存储位置 | 生命周期 | 使用场景 |
|----------|----------|----------|----------|
| 会话内存 | 运行时内存 | 单次会话 | 当前对话上下文 |
| 每日记忆 | memory/YYYY-MM-DD.md | 永久 | 每日事件记录 |
| 长期记忆 | MEMORY.md | 永久 | 重要信息、偏好设置 |
| 技能知识 | skills/*/SKILL.md | 版本控制 | 领域专业知识 |

### C. 错误处理机制

```
工具调用失败
├── 网络错误 → 自动重试（最多3次）
├── 权限错误 → 向用户报告并请求授权
├── 数据错误 → 使用备用数据源或告知用户
└── 未知错误 → 记录日志并优雅降级
```
