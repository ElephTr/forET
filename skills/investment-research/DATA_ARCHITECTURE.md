# 全球市场数据架构设计

## 第一性原理：可信度 = 多源交叉验证 + 实时性 + 权威性

## 数据源分层架构

### Tier 1: 官方/交易所直连（最高可信度）
- 交易所官方API（延迟最低，数据最准）
- 央行/统计局官方发布
- 监管机构披露

### Tier 2: 专业数据供应商（高可信度）
- Bloomberg, Refinitiv
- Wind, 同花顺iFinD
- Alpha Vantage, Quandl

### Tier 3: 聚合平台（中等可信度，用于交叉验证）
- Yahoo Finance
- Investing.com
- 东方财富

### Tier 4: 另类/情绪数据（辅助验证）
- 社交媒体情绪
- 搜索趋势
- 链上数据（加密）

## 数据验证机制

### 1. 多源交叉验证
```
同一指标从3+来源获取
如果偏差>阈值，标记为异常，人工复核
```

### 2. 时间一致性检查
```
检查数据时间戳
剔除延迟过大的数据源
```

### 3. 逻辑一致性检查
```
指数成分股加权 vs 指数报价
期货升贴水 vs 现货价格
```

## 故障转移策略

1. 主源失败 → 自动切换到备用源
2. 多源数据冲突 → 标记异常，使用多数一致值
3. 全部失败 → 使用缓存数据，标记为 stale

## 可信度评分

每个数据点附带可信度分数：
- 100: 官方交易所实时数据
- 90: 专业供应商实时数据
- 80: 聚合平台实时数据
- 70: 延迟数据（15分钟+）
- 60: 单源数据（未验证）
- 50: 估算/模型数据

## 实现优先级

P0: 核心市场数据（美股、A股、港股、外汇、商品）
P1: 宏观数据（利率、通胀、就业）
P2: 情绪数据（VIX、资金流向、情绪指标）
P3: 另类数据（链上、搜索趋势、社交情绪）
